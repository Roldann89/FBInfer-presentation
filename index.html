01. Presentación

¿Qué es FBInfer?

Analizador estático de código para encontrar errores graves antes de la compilación o despliegue.

  • Prevención Activa: Evita fallos que detienen apps (punteros nulos, etc.).
  • Integración en CI/CD: Valida cambios en minutos en servidores de integración.
  • Multilenguaje: Soporte completo para Java, C, C++ y Objective-C.

03. Innovación Teórica

Análisis Composicional y Bi-Abducción

Clave para el escalado masivo mediante razonamiento matemático incremental de trozos de código aislados.

1
Análisis Local (Composicional)

Infer analiza cada función de manera aislada sin revisar todo el programa.

2
Pre y Postcondiciones

Determina qué necesita la función (Pre) y qué altera al terminar (Post).

3
Bi-Abducción

Deduce automáticamente las partes de memoria que faltan para conectar funciones con seguridad.

Análisis Global vs. Composicional

Enfoque Tradicional
Lento y no escalable

Evalúa todo el programa en conjunto. Inviable para millones de líneas.

Enfoque de Infer
Lineal y Veloz

Divide el análisis por métodos. Usa lógica formal para conectar resultados.

04. Capacidades

Errores Críticos que Detecta

Identificación de fallos lógicos graves que comprometen la estabilidad del software.

Punteros Nulos

Referencias vacías que detienen la app.

Object o = null;
o.toString(); // ❌ Bug
JavaC/C++

Fugas de Memoria

Asignaciones heap sin liberar.

int* p = malloc(10);
// ❌ Sin free(p)
C/C++Obj-C

Fugas de Recursos

Streams y sockets que no se cierran.

File f = open(...);
// ❌ Sin close()
JavaC/C++

Thread Safety

Condiciones de carrera con hilos.

// Acceso simultáneo
contador++; // ❌ Inseguro
Java

Facilidad de Instalación por Plataforma

Nativo en entornos basados en Unix. En Windows requiere capas de compatibilidad.

Virtualizado

07. Demo Práctica

Ejemplo de Uso y Reporte de Bugs

Detección de errores en código Java y C. Alterna en las pestañas para ver las demos:

Ejemplo.java
import java.io.*;
 
public class Ejemplo {
    public int obtenerLongitud(String texto) {
        String miTexto = null;
        if (texto != null) miTexto = texto;
        
        // BUG 1: Puntero Nulo (puede explotar)
        return miTexto.length(); 
    }

    public void escribir(String ruta, String d) throws Exception {
        // BUG 2: Fuga de Recurso (FileOutputStream)
        FileOutputStream fos = new FileOutputStream(ruta);
        fos.write(d.getBytes());
        fos.close(); 
    }
}

leak.c
#include <stdlib.h>
#include <stdio.h>
 
void procesar(int cond) {
    // Se reserva memoria en el heap
    int *buffer = (int *)malloc(100 * sizeof(int));
    
    if (cond > 50) {
        printf("Error fatal\n");
        // BUG: Salimos sin liberar con free()
        return; 
    }
    
    free(buffer); // Proceso normal
}

08. Resumen

Conclusiones del Análisis

Evaluación del impacto en la ingeniería de software moderna:

Ventajas
  • Precisión: Rigor matemático que reduce falsos positivos.
  • Composicionalidad: Análisis modular de gran velocidad.
  • Integración: Automatización en flujos de CI/CD.
Limitaciones
  • Sin Windows nativo: Exige uso de Docker/WSL2.
  • Curva técnica: Explicación de trazas de error compleja.
  • Dependencia: Intercepta compiladores específicos.
